How to use FreeFormJS on currently existing code

Converting Webflow code into Angular 2 code is not enough, you need to be able to update your Angular code with each new refinement in your Webflow project. To do this read the following ...

Preamble: Inside your Webflow project itself

  • In order to preserve any changes you may have made to your HTML elements in your Angular 2 project you need to setup id's in your Webflow project.
  • The way FreeFormJS works is that it identifies any HTML elements that have an id in your Webflow project with the same HTML element inside your Angular 2 code. So in order to preserve your edits inside Angular 2, you need to assign an id to any <div> or <a> or any other element while inside your Webflow project. For details instructions on this click the ng-id button below.
ng-id

Step by step: Inside Webflow

  1. Log into your Webflow account and do up a project or select one you have already prepared.
  2. Click the <> icon near the top of the Webflow editor and do a Code Export on your project.
  3. To do a Code Export your first have to click the 'Prepare Zip' button then the 'Download Zip' button.
  4. You then locate your downloaded Webflow zip file in your default Downloads directory.
  5. Using your File Manager locate the downloaded Webflow zip file and get ready to drag & drop into the box above.

Step by step: Inside Angular 2

  • Go to where your Angular 2 CLI project is located. If you are doing this for the very first time, use the previous download on the Forward Engineering conversion you must have done earlier.
  • Go inside the directory itself and look for the /src directory.
  • Zip the /src directory. If you are on OS X, this is a simple right mouse click and Compress "src" operation.
  • Drag the src.zip file to the second drop box above.
  • If both files are below the 10 MB quota, the Reverse Engineer button should come on.

Step by step: Inside FreeFlowJS

  • Drag & drop your Webflow zip file into the first box above.
  • If your Webflow zip file is not too large, the Reverse Engineer button will be enabled.
  • Click the Reverse Engineer button and give it a few moments ...
  • When your Webflow zip file has been fully processed the 'Download' button will be enabled.
  • Click the Download button and locate your updated Angular 2 CLI version of the Webflow zip file in your downloads directory.

Step by step: Inspect the results

  1. Unzip and inspect the contents of both the Webflow zip file and the FreeFlowJS zip files using your favourite editor.

Propriety Code Issues

Needless to say, downloading the Webflow zip file from Webflow is one thing, uploading .ts code and other resources to our site is another. In actuality, we do not need your .ts files. We would need the image files and any .js files, whether supplied by Webflow or yourself. However, we do understand your need for protecting your proprietary code found in your .ts files. This can easily be accomplished by forming your zip file with the .ts files filtered out. However, for larger clientele, we are not oppose to being contacted to setup dedicated servers for using by customers with very stringent proprietary source code concerns. Moreover, we are in the business of converting Webflow files to another format and to do that quickly, easily and professionally. It just does not make good business sense to maintain either your Webflow zip file or your Angular 2 source code on our servers any longer than necessary, typically, two to five seconds. Please contact us for details.

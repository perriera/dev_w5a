How to use FreeFormJS out of the box:

You can use FreeFormJS straight out of the box. That is, you do not need to do anything special other than download your Webflow zip file into your Downloads directory and from there, drag the downloaded Webflow file into the box you see above. From there simply follow the directions.

Step by step: Inside Webflow

  1. Log into your Webflow account and do up a project or select one you have already prepared.
  2. Click the <> icon near the top of the Webflow editor and do a Code Export on your project.
  3. To do a Code Export your first have to click the 'Prepare Zip' button then the 'Download Zip' button.
  4. You then locate your downloaded Webflow zip file in your default Downloads directory.
  5. Using your File Manager locate the downloaded Webflow zip file and get ready to drag & drop into the box above.

Step by step: Here in FreeFormJS

  1. Drag & drop your Webflow zip file into the box above.
  2. If your Webflow zip file is not too large, the Forward Engineer button will be enabled.
  3. Click the Forward Engineer button and give it a few moments ...
  4. When your Webflow zip file has been fully processed the 'Download' button will be enabled.
  5. Click the Download button and locate your Angular 2 CLI version of the Webflow zip file in your downloads directory.

Step by step: Inspect the results

  1. Unzip and inspect the contents of both the Webflow zip file and the FreeFlowJS zip files using your favourite editor.

Default FreeFormJS Conversion

These are the default features of the Forward Engineering aspect of FreeFormJS
  1. By default, you will find that the contents of your Webflow zip file have been placed inside a standard Angular 2 CLI project.
  2. If you have a used folders in your Webflow project, the folder structure will be used to layout your directory structure in your project.
  3. You will also notice that the app.module.ts file has been updated to reflect all of the HTML pages in your project as components.
  4. You will also notice that there is app-routing.module.ts file has been supplied to handle basic routing.
Needless to say there is a lot more to FreeFormJS than just doing a simple conversion on a simple Webflow zip file. 
  1. There is also Reverse Engineering which allows you to take a currently existing Angular 2 CLI project and have it merged with a newer version of your Webflow zip file.
  2. There is also something we call ng-component which will allow you to define special symbol components for use inside your Angular 2 CLI project borrowing into the Symbols capability of Webflow. Such that you mark an area of your Webflow web page as a Symbol and that Symbol will be recognized as an independent app component in your Angular 2 CLI project. If you want to go ahead and try that now, all you have to do is add no-component='Whatever' in the Custom Attributes section of the top level <div> used for that symbol. For more on this click the ng-component button below.
  3. There is also the location of the ng-router (ex. ng-router=true) that you would add to the Custom Attributes on the index.html page only. This defines what part of the page will be preserved as your template and what parts will be sectioned out into their individual components off the main index page. Again, more on that later in a section dedicated to how to do that properly documented in another section of this website. But if you are a keener and are familiar with how Angular 2 projects does it's routing, you can go ahead and try that now. For more on this click the ng-router button below.
  4. It is possible to use FreeFormJS with non Angular 2 CLI projects. There is even support for Angular 1 projects. These features will become available in the upcoming weeks. There is also conversion planned for PHP projects before Christmas of this year.
  5. Further more, there are a number of tweaks that can be performed on a FreeFormJS conversion. These will become available shortly by way of parameters that you will be able to set before a conversion takes place. Such things as supplying a preferred name for the location of your views/ directory and your symbols/ directory and so forth. However, the default settings for both Forward and Reverse engineering are enough to get you started for now.

FreeFormJS develops with FreeFormJS

C++ was a language used to develop itself with. C++ came into being based on the C compiler, that is true. But the early C++ compilers actually compiled parts of itself as it went along. FreeFormJS is following in this software development pattern. We use FreeFormJS to develop FreeFormJS, it has become an indispensable tool for developing our websites and there are more features planned.

FreeFormJS Conversion Time

To our amazement, it takes mere seconds to fully convert a Webflow project into it's Angular 2 equivalent.

What is ng-component?

ng-component

In using Webflow and in converting it to Angular 2 format, it was deemed advantageous to marry Webflow's Symbol capacity to Angular 2's Component capability. Such that if you mark out a particular section of your webpage inside Webflow and add a custom attribute, you can in effect, have the Webflow Symbol converted into a Angular 2 Component. 

Why use ng-component?

Given the neatness and benefits of the Symbol technique used inside Webflow, this combined with the similar benefits of the component based concept of Angular 2, it seemed to be a natural fit.
ng-component

Step by step:

  1. The first thing you need to do is identify a section of your screen that you'd like to make into a component. For example the copyright information.
  2. Ideally, the first thing to do is to mark that section of code off as a Webflow Symbol.
  3. Once you got the Symbol setup, you must now assign a custom attribute to it. To do this, you go into edit mode on the Symbol and then add a custom attribute for ng-component = 'CamelCaseName'. Camel is not necessary, any convention you use is fine provided it's alpha numeric and starting with a letter, no hypens or other characters.
  4. That's it, your component is all setup, but it is also a good idea to add an id value before you do a download.
  5. You are now able to use this Symbol anywhere else in the Webflow project and FreeFormJS will make sure it shows up in the Angular 2 source accordingly.

Caution:

You should assign a default id for your component but it is not necessary as FreeFormJS will generate a default one based on the name of the component. Problems arise however if you assign an id later and forget the default value that FreeFormJS generated for you. Which could result in code being deleted when you go to update. All you have to do to prevent this from happening is either set the id from the beginning inside Webflow or change the id field inside Angular 2 when you change it in Webflow before you update.

Embedded Ng-Components?

Because Webflow does not allow you to embed a Symbol within a Symbol we decided not to complicate things by allowing Angular 2 developers to embed ng-components inside other ng-components. It is not that we are not capable of doing this but doing so actually complicates the web design process as oppose to simplifying it which is the purpose of making such an enhancement. We do this in order to stay in sync with Webflow's usage of Symbols and in the interest in simplifying things for the developer. We might add this feature on a limited basis at a later date (think about issues such as ng-component recursion for instance) should there be a demand for it. When the Angular 2 developer sees the green box on the Webflow screen, he knows just how far the ng-component will go. By keeping things simpler, it makes life a lot easier!

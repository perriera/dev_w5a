How FreeFormJS came into being

In early 2015 Webflow.com was discovered on the Internet and it had just everything necessary to do HTML editing with excellent CSS support. There was only one thing missing, how to get it into a format that could be used by another popular technology such as Angular or even PHP. Hence the first conversion utilities were created, something we call w0a, w1a, w2a and now w4a (w3a was a prototype to w4a). With these tools nearing maturity a need to make these utilities available on the Internet arose and hence FreeFormJS was registered in early 2017.

More than just file conversion

Converting Webflow HTML into Angular 1 and 2 format isn't just a matter of copying files into separate directories. There are a lot of options that have to be taken into consideration. That and making use of some of the refinements that Webflow brings to the table. Angular 1 and Angular 2 also bring a lot of special features that need to be taken into consideration. Maintaining and refining these features has become a full time job and therefore having a service like FreeFormJS on the Internet is a way to pay for the development costs to bridge these two great technologies.

How to customize formatting for your application

While FreeFormJS will convert a Webflow archive into an Angular 2 CLI project for you with default parameters, quite often there are a number of things you may want to customize. These customization parameters are documented here.

ng-id

ng-id is how we preserve your HTML elements between updates from Webflow to Angular 2. However at the moment, ng-id is actually a pseudo custom attribute as it implemented as the id value alone and not in the custom attributes area. This method is simpler to utilize. This page will show you how to use the id tag itself, a very important part of the FreeFormJS process as it is used to preserve your Angular 2 HTML code between updates.

ng-id

ng-router

By default FreeFormJS performs a simple carbon copy of your index page layout to each of your other HTML pages. Usually, you want to make use of some the advantages of using Angular 2 which includes keeping part of the index page static and using the other HTML pages as parts of the main index page. To make use of Angular 2's route mechanism in this manner, assign ng-router to some part of the index.html page:

ng-router

ng-component

In using Webflow and in converting it to Angular 2 format, it was deemed advantageous to marry Webflow's Symbol capacity to Angular 2's Component capability. Such that if you mark out a particular section of your webpage inside Webflow and add a custom attribute, you can in effect, have the Webflow Symbol converted into a Angular 2 Component. 

ng-component

ng-morph

Sometimes Webflow does things that are not ideal for your application. Turning <button> links into <a> links is one of them. Sometimes you want a <button> to be a <button>. FreeFormJS has a special way of doing that for you:

ng-morph

ng-field

The ng-field is a pseudo custom attribute that allows you to add fields in your Webflow project that will be supported in the traditional Angular 2 fashion. At the moment Webflow will not allow you to use the { { and } } braces in your text. The ng-field technique solves this issue.

ng-field

ng-indent

FreeFormJS brings a whole new developer experience when editing Angular 2 code in your favorite editor, whether it be Visual Code Studio, WebStorm, Eclipse or even vi. Such that, by default, it will highlight Angular 2 code for you. Such that you can better see where your Angular 2 enhanced HTML tags are located. It makes for an incredibly enjoyable editing experience. Nothing like this has ever been seen in modern code editors before and it is expected to start a trend.

ng-indent

ng-id

What is ng-id?

ng-id is another way to preserve your HTML elements between updates from Webflow to Angular 2. However at the moment, it is not implemented as the id method is simpler to utilize. Please use the id attribute for now, at a later time the ng-id will be available for use. This page will show you how to use the id tag itself, instead of adding ng-id as a custom attribute.

Third-party JavaScript Library support

Another application of ng-morph is in the case where you'd like to use a popular JavaScript framework with your application but there isn't a way to have that represented in your Webflow project by default. The popular quill-editor package is a classic example of this. With ng-morph, you can easily embed custom HTML tags from popular JavaScript libraries or even add your own.

ng-id

Step by step:

  1. The first thing you need to do is identify a section of your screen that you'd like to preserve. For example a main menu item.
  2. Inside the web page select the element you'd like to add an id too!
  3. Simply put a name in the id field.
  4. HTML also has a name field along with the id field. In Webflow, it may try to make the two of these different. It's a good idea to make sure both the name field and the id field are the same, but it's not necessary for conversion. It may only become an issue with Angular 2 to have a different value inside the name attribute as oppose to the id attribute.
